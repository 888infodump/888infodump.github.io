Hashcat Detailed Guide

Introduction to Hashcat

Hashcat is a powerful password recovery tool that supports various hashing algorithms and offers multiple attack modes.

Launching Hashcat

Open your terminal and navigate to the directory where Hashcat is installed. To see the basic usage and options, run: 

hashcat -h

Supported Hash Types

Identify the hash type you are working with. Use the following command to list supported hashes: 

hashcat --help | grep hash

Preparing Your Hash File

Create a text file containing the hashes you want to crack, one per line. For example: 


            5f4dcc3b5aa765d61d8327deb882cf99

            6dcd4ce23d88e2ee9568ba546c007c63

Selecting an Attack Mode

Hashcat supports several attack modes. Common modes include:

  • Straight (dictionary attack)
  • Combination (combine words from multiple lists)
  • Brute-force (try all possible combinations)
Set the mode with the -a option: 

hashcat -a 0 -m <hash-type> <hash-file> <wordlist>

Using a Wordlist

To crack hashes using a wordlist: 

hashcat -m <hash-type> <hash-file> <path-to-wordlist>

Example with a common wordlist: 

hashcat -m 0 myhashes.txt /usr/share/wordlists/rockyou.txt

Using Rules for Enhanced Cracking

Apply rules to modify the wordlist entries: 

hashcat -m <hash-type> <hash-file> <wordlist> -r <rules-file>

Example: 

hashcat -m 0 myhashes.txt /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/best64.rule

Brute-Force Attack

For brute-force cracking, specify a mask: 

hashcat -m <hash-type> <hash-file> -a 3 ?l?l?l?l

This example tries all combinations of four lowercase letters.

Using Hashcat with GPUs

Hashcat can leverage GPUs for faster cracking. Ensure you have the necessary drivers installed. Check available devices: 

hashcat -D

Monitoring Progress

During cracking, Hashcat displays the progress, including the current speed and estimated time.

Pausing and Resuming

You can pause the cracking process with 

s
and resume it later using: 

hashcat --restore

Outputting Results

Cracked passwords will be saved to a .pot file. You can specify a different output file with: 

hashcat -m <hash-type> <hash-file> <wordlist> --outfile=<output-file>

Using Hashcat with the API

Hashcat also offers an API for integration with other tools and scripts.

Getting Help and Documentation

Refer to the official Hashcat documentation for in-depth features and options:
